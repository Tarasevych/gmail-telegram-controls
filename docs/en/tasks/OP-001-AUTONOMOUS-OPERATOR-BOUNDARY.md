# OP-001: Autonomous operator boundary

- Status: active permanent rule
- Date: 2026-07-19
- Applies to: Versie 1 and every subsequent sequential release

## Authorization

The project owner authorizes Codex to execute routine operational steps autonomously when required to implement, test, release, and maintain an approved task:

- control the browser and PC;
- use credentials already stored by the system password manager;
- complete Google sign-in and OAuth consent for the specific account already identified by the owner in the private checkpoint;
- start required elevated processes and install necessary tools;
- run local tests, staging, deployment, Git commits, and normal pushes;
- fix technical defects and preserve verified evidence.

## Mandatory owner intervention

- CAPTCHA;
- OTP that cannot be obtained safely through an available programmatic channel;
- a purchase, payment, or other financial commitment;
- a physical action;
- an ambiguous unsafe, destructive, or materially irreversible choice.

## Permanent safeguards

- Never publish passwords, tokens, cookies, keys, or recovery codes.
- Never mix Gmail accounts, Telegram zones, or parallel runtime lanes.
- Never alter random email or unrelated user data.
- No force-push, reset, deletion, or history rewriting without an explicitly verified need.
- Use administrative privileges only when required by the concrete step.

This file defines the durable operating boundary. Specific accounts, secrets, and retrieval methods remain only in private protected context.
