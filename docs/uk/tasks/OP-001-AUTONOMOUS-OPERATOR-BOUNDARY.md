# OP-001: Межа автономного виконання

- Статус: чинне постійне правило
- Дата: 2026-07-19
- Застосування: Versie 1 та всі наступні послідовні випуски

## Повноваження

Власник проєкту дозволяє Codex самостійно виконувати операційні кроки, потрібні для реалізації, тестування, випуску та підтримки погодженої задачі:

- керувати браузером і ПК;
- використовувати облікові дані, вже збережені штатним password manager;
- виконувати вхід у Google та OAuth consent для конкретного акаунта, який власник уже визначив у приватному checkpoint;
- запускати потрібні процеси з підвищеними правами та встановлювати необхідні інструменти;
- виконувати локальні тести, staging, deployment, Git commit і звичайний push;
- виправляти технічні дефекти та фіксувати перевірені результати.

## Коли потрібне втручання власника

- CAPTCHA;
- OTP, який неможливо безпечно отримати доступним програмним каналом;
- покупка, платіж або інше фінансове зобов'язання;
- фізична дія;
- неоднозначний небезпечний, руйнівний або суттєво незворотний вибір.

## Незмінні обмеження

- Не публікувати паролі, токени, cookies, ключі або recovery-коди.
- Не змішувати Gmail-акаунти, Telegram-зони або паралельні runtime-потоки.
- Не змінювати випадкові листи чи сторонні дані.
- Не використовувати force-push, reset, видалення або переписування історії без явно перевіреної потреби.
- Використовувати адміністративні права лише коли вони потрібні конкретному кроку.

Цей файл визначає постійну операційну межу. Конкретні акаунти, секрети та методи їх отримання зберігаються лише у приватному захищеному контексті.
