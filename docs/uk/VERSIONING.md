# Версіонування та незмінність випусків

Дата набуття чинності: **2026-07-19**.

## Чому відлік починається з Versie 1

Історія використовувала паралельні продуктові та технічні числа v27-v47, через що назва папки, продукту, web app і Apps Script могла відрізнятися. Від Versie 1 продукт має одну послідовну шкалу: **Versie 1, Versie 2, Versie 3, ...**. Пропуски, повернення до старого номера та паралельні продуктові випуски заборонені.

## Правила

1. Нова робота вноситься тільки в єдину активну гілку наступної Versie.
2. Після acceptance створюються незмінні `release/build-NNN-YYYY-MM-DD` і tag `build-NNN-YYYY-MM-DD`.
3. Попередній release не редагується. Виправлення стає частиною наступної Versie і посилається на проблему попередньої.
4. `main` містить останню підтверджену публічну Versie та її GitHub Pages.
5. Web app, Telegram menu label, release article, Git commit/tag і production deployment мають посилатися на одну Versie.
6. Внутрішній Apps Script immutable number не можна перезапустити. Він зберігається як технічний доказ у таблиці зіставлення, але не є продуктовим номером.
7. Кожна стаття Versie повторює кумулятивну історію, актуальні можливості, відкриті проблеми й результати тестів та посилається на попередню статтю.
8. Hotfix також отримує наступну послідовну Versie, а не змінює старий release.
9. До release заборонено заявляти `Released`, доки production, Telegram і контрольні функції не перевірені.

## Гілки

- `codex/versie-001-2026-07-19`: єдина активна робоча гілка Versie 1.
- `release/versie-001-2026-07-19`: буде створена тільки після успішного production acceptance.
- `main`: останній підтверджений публічний стан.
- Старі `codex/neuroinclusive-*` та технічні гілки залишаються read-only історією.
