## Нові зміни Versie 1

- GT-001: усунуто подвійне сканування owner Gmail через legacy та OAuth lanes без злиття чужих connection records.
- GT-002: Google повертає на neutral GitHub Pages callback, який очищає URL і передає Apps Script лише одноразові `state/code` через credentialless POST.
- GT-003: desktop/mobile header використовує Google profile photo з fallback.
- GT-004: Google OAuth відкривається одним кліком; fallback з'являється лише при блокуванні.
- GT-005: stale/inactive account IDs більше не збільшують unified count.
- GT-009: виправлено однину accessibility label.
- GT-011: Telegram settings містять нативні callback-кнопки для one-click перемикання та короткий GitHub launcher для додавання Gmail.
- GT-012: браузер більше не навігується на Apps Script callback, тому Google multi-login не може переписати URL у несправний `/macros/u/N/`.
- GT-013: owner `/settings` відкриває нативний account menu замість fallback.
- GT-014: Telegram production menu використовує commands, не Apps Script web_app.
- GT-015–GT-017: окремо зафіксовані duplicate display records, системний Telegram Open Link і legacy Mini App deep links; Gmail-записи не видалялися й не змішувалися.
- OAuth state v2 має TTL, одноразове споживання й точну прив'язку до Telegram user/chat та поштової зони; після успіху бот надсилає оновлене account menu.
- Gmail OAuth prompt зменшено до `select_account`; новий consent усе одно контролює Google і проходить лише власник.

## Перевірені докази

- Apps Script dashboard: один trigger `checkNewMail_`; дубль другого notifier project не підтверджено.
- Локальні контракти Versie 1: **432/432 passed** для candidate commit b5a5452.
- Release inventory 2026-07-21 PreflightOnly: stable v50, HEAD відповідає stable v50, current staging 0, лишається один legacy staging deployment, v55 readyToStage; v55 deployment або production promotion не виконувалися.
- GitHub Pages relay не зберігає query, storage або cookies і надсилає callback тільки точному stable Apps Script endpoint.
- Причину `/macros/u/N/` підтверджує [офіційний Apps Script troubleshooting](https://developers.google.com/apps-script/guides/support/troubleshooting): multi-login не підтримується для Apps Script web apps.
- OAuth redirect URI збережено в Google OAuth client і прочитано назад без розбіжностей.
- `setupTelegramControls` завершився в Apps Script editor; live Telegram menu і `/settings` перевірені.
- Реальний OAuth launcher дійшов до Google «Спеціалісти Google не перевіряли цей додаток» без Apps Script Drive error; consent не натиснуто.

## Що ще не доведено

- Потрібен user-controlled Google consent для правильного контрольного Gmail-акаунта; вкладка залишена на цьому gate.
- Потрібен real-time тест від контрольованого листа до однієї Telegram-картки.
- Потрібен staging visual/readback для avatar, direct OAuth і account count.
- Versie 1 ще не tagged і не має immutable release branch, бо callback success та real-time mail acceptance не завершені.
