## Навіщо Versie 1

Це перша версія нової послідовної продуктової шкали. Старі v27-v47 не видаляються: вони залишаються технічними й дослідницькими ланками, але більше не змагаються з продуктовим номером. Versie 1 збирає весь підтверджений функціонал попередньої лінії та додає виправлення поточного інциденту.

## Нові зміни Versie 1

- GT-001: усунуто подвійне сканування owner Gmail через legacy та OAuth lanes без злиття чужих connection records.
- GT-002: Google повертає на neutral GitHub Pages callback, який очищає URL і передає Apps Script лише одноразові `state/code` через credentialless POST.
- GT-003: desktop/mobile header використовує Google profile photo з fallback.
- GT-004: Google OAuth відкривається одним кліком; fallback з'являється лише при блокуванні.
- GT-005: stale/inactive account IDs більше не збільшують unified count.
- GT-009: виправлено однину accessibility label.
- GT-011: Telegram settings містять нативні callback-кнопки для one-click перемикання та короткий GitHub launcher для додавання Gmail.
- GT-012: браузер більше не навігується на Apps Script callback, тому Google multi-login не може переписати URL у несправний `/macros/u/N/`.
- GT-013: owner `/settings` відкриває нативний account menu замість fallback.
- GT-014: production menu знову має Web App `📬 Пошта · Versie 1`, але відкриває neutral GitHub Pages bridge, а не прямий Apps Script URL.
- GT-015–GT-017: окремо зафіксовані duplicate display records, системний Telegram Open Link і legacy Mini App deep links; Gmail-записи не видалялися й не змішувалися.
- OAuth state v2 має TTL, одноразове споживання й точну прив'язку до Telegram user/chat та поштової зони; після успіху бот надсилає оновлене account menu.
- Gmail OAuth prompt зменшено до `select_account`; новий consent усе одно контролює Google і проходить лише власник.
- GT-010: token refresh захищено per-connection lease і короткими claim/commit/release locks без network I/O під lock.
- GT-018/GT-019: bounded realtime lane працює перед frozen backlog; ручний `/check` використовує той самий dedupe ledger.

## Що ще не доведено

- Fresh OAuth account choice/consent/callback для нового Gmail-акаунта не перевірено; наявні сесії не доводять цей шлях.
- Незалежний real-time fan-out із другого підключеного акаунта не перевірено.
- Перше production-відкриття один раз лишилося на skeleton понад 15 секунд і завантажило mailbox після refresh (GT-021).
- `clasp logs` заблокований відсутнім GCP project ID у локальному config; project identity не вгадувалася (GT-022).
- Versie 1 ще не tagged як завершений product release; створення Versie 2 потребує окремого точного owner-наказу.
