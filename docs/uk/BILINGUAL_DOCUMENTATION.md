# Двомовна документація GitHub

English mirror: [docs/en/BILINGUAL_DOCUMENTATION.md](../en/BILINGUAL_DOCUMENTATION.md).

## Обов'язкове правило

Кожна опублікована документаційна сторінка має українську й англійську версії з однаковим відносним шляхом:

- `docs/uk/<path>.md`;
- `docs/en/<path>.md`.

Якщо створюється нова сторінка, обидва файли створюються одночасно через `tools/create_bilingual_page.py`. Якщо змінюється одна мовна версія, відповідна друга версія має бути змістовно оновлена в тому самому commit range.

Кореневі або службові Markdown-сторінки, які не мають мовних каталогів, мають містити повні секції `Українською` та `English` в одному файлі.

## Автоматичний контроль

`tools/check_bilingual_docs.py` перевіряє:

- однаковий набір Markdown-шляхів у `docs/uk` і `docs/en`;
- відсутність порожніх мовних сторінок;
- одночасну зміну обох мов для кожного зміненого шляху;
- двомовні маркери в кореневих спільних сторінках.

Workflow `.github/workflows/bilingual-docs.yml` запускає цю перевірку для push і pull request. Одностороння сторінка або зміна завершує workflow помилкою й не вважається готовою до публікації.

Автоматична перевірка не підміняє переклад копією іншої мови. Обидві версії мають передавати однаковий зміст природною мовою.
