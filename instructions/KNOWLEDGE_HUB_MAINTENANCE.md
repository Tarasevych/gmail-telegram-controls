# Підтримка knowledge hub / Knowledge-hub maintenance

Source request: `REQ-0003`.

<!-- lang:uk -->
## Українською

## Routine-порядок читання

1. Прочитати поточний `REQ-ID` у `Запити`.
2. Прочитати `Інструкції` та лише релевантне `Повноваження`.
3. Відкрити `docs/uk/knowledge-hub/README.md` активної Versie.
4. Відкрити `MASTER_ROADMAP.md` і лише один потрібний тематичний реєстр.
5. Відкрити актуальний `docs/uk/verification-reports/INDEX.md`, потрібний report і конкретний claim.
6. Звірити відповідний поточний `PROJECT`, `ROADMAP` або `ISSUES` перед live-твердженням.

Source dossiers і три великі deep-research reports не перечитуються для routine-роботи. До них повертаються лише за конкретним provenance gap, unresolved conflict, новим аудитом або обов'язковим pre-version review.

## Порядок оновлення

- Новий запит спочатку фіксується в `Запити`.
- Зміна вноситься до релевантної мовної пари й canonical `KH-*` item.
- `TRACEABILITY`, `catalog.json` і `manifest.json` оновлюються, якщо змінюється mapping або додається нове джерело.
- Report-derived твердження не стає live-verified без нового доказу.
- Зміна verification status відбувається лише через трасований factual verification report; E1/E2 не підвищується до runtime або production без E4/E5.
- Instruction candidates стають чинними лише в `Інструкції`.
- Permission candidates не є дозволами; зміна `Повноваження` потребує прямого рішення власника.
- Нова Versie не створюється без окремого прямого наказу.

## English

## Routine reading order

1. Read the current `REQ-ID` on `Запити`.
2. Read `Інструкції` and only the relevant `Повноваження` record.
3. Open `docs/en/knowledge-hub/README.md` on the active Versie.
4. Open `MASTER_ROADMAP.md` and only one relevant thematic register.
5. Open the current `docs/en/verification-reports/INDEX.md`, the applicable report, and the specific claim.
6. Reconcile the applicable current `PROJECT`, `ROADMAP`, or `ISSUES` before making a live claim.

Source dossiers and the three large deep-research reports are not reread for routine work. Return to them only for a specific provenance gap, unresolved conflict, new audit, or mandatory pre-version review.

## Update order

- Record a new request on `Запити` first.
- Update the relevant language pair and canonical `KH-*` item.
- Update `TRACEABILITY`, `catalog.json`, and `manifest.json` when a mapping changes or a source is added.
- A report-derived claim does not become live-verified without new evidence.
- A verification-status change occurs only through a traced factual verification report; E1/E2 is not promoted to runtime or production without E4/E5.
- Instruction candidates become standing only on `Інструкції`.
- Permission candidates grant no authority; changing `Повноваження` requires a direct owner decision.
- A new Versie still requires a separate direct order.
