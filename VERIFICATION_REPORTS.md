# Незалежні factual verification reports / Independent factual verification reports

<!-- lang:uk -->
## Українською

Актуальний двомовний індекс незалежних factual verification reports міститься в активній [Versie 1](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/uk/verification-reports/INDEX.md).

Найновіший звіт [VR-003](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/uk/verification-reports/reports/VR-003/README.md) класифікує 32 атомарні claims із двох приватних session corpora за primary Git/test evidence. [Machine-readable ledger](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/verification-reports/VR-003/claims.json) зберігає status, scope, evidence grade, provenance, dependencies, conflicts, sensitivity і limitations кожного твердження. VR-001 залишається історичним 245-claim baseline.

- `E1/E2` доводить Git/file або static implementation, а не runtime чи production.
- Runtime потребує `E4`, production acceptance потребує `E5`.
- Recommendation не є реалізованою функцією, standing instruction або owner-granted permission.
- `Повноваження` не змінено через VR-003.
- Runtime, deployment і `Versie 1` release identity не змінено через `REQ-0012`.

Verification framework: `REQ-0004`. Source request: `REQ-0012`.

## English

The current bilingual index of independent factual verification reports is on active [Versie 1](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/en/verification-reports/INDEX.md).

The latest report, [VR-003](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/en/verification-reports/reports/VR-003/README.md), classifies 32 atomic claims from two private session corpora against primary Git/test evidence. The [machine-readable ledger](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/verification-reports/VR-003/claims.json) preserves each claim's status, scope, evidence grade, provenance, dependencies, conflicts, sensitivity, and limitations. VR-001 remains the historical 245-claim baseline.

- `E1/E2` proves Git/file or static implementation, not runtime or production.
- Runtime requires `E4`; production acceptance requires `E5`.
- A recommendation is not an implemented feature, standing instruction, or owner-granted permission.
- `Повноваження` was not changed through VR-003.
- Runtime, deployment, and the `Versie 1` release identity were not changed through `REQ-0012`.

Verification framework: `REQ-0004`. Source request: `REQ-0012`.
