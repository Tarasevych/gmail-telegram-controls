# Незалежні factual verification reports / Independent factual verification reports

<!-- lang:uk -->
## Українською

Актуальний двомовний індекс незалежних factual verification reports міститься в активній [Versie 1](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/uk/verification-reports/INDEX.md).

Поточний звіт [VR-001](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/uk/verification-reports/reports/VR-001/README.md) класифікує 245/245 `KH-*` claims за первинними Git-доказами та безпечними локальними тестами. [Атомарний ledger](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/uk/verification-reports/reports/VR-001/CLAIMS.md) зберігає status, scope, evidence grade, походження, залежності, конфлікти, sensitivity і limitations кожного твердження.

- `E1/E2` доводить Git/file або static implementation, а не runtime чи production.
- Runtime потребує `E4`, production acceptance потребує `E5`.
- Recommendation не є реалізованою функцією, standing instruction або owner-granted permission.
- `Повноваження` не змінено через `VR-001`.
- Runtime, deployment і `Versie 1` release identity не змінено через `REQ-0004`.

Source request: `REQ-0004`.

## English

The current bilingual index of independent factual verification reports is on active [Versie 1](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/en/verification-reports/INDEX.md).

The current report, [VR-001](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/en/verification-reports/reports/VR-001/README.md), classifies 245/245 `KH-*` claims against primary Git evidence and safe local tests. The [atomic ledger](https://github.com/Tarasevych/gmail-telegram-controls/blob/codex/versie-001-2026-07-19/docs/en/verification-reports/reports/VR-001/CLAIMS.md) preserves each claim's status, scope, evidence grade, provenance, dependencies, conflicts, sensitivity, and limitations.

- `E1/E2` proves Git/file or static implementation, not runtime or production.
- Runtime requires `E4`; production acceptance requires `E5`.
- A recommendation is not an implemented feature, standing instruction, or owner-granted permission.
- `Повноваження` was not changed through `VR-001`.
- Runtime, deployment, and the `Versie 1` release identity were not changed through `REQ-0004`.

Source request: `REQ-0004`.
