# P-002: Google-вхід, OAuth і збережені облікові дані / Google sign-in, OAuth, and stored credentials

- Статус / Status: active
- Джерело / Source: пряма вказівка власника / direct owner instruction
- Дата / Date: 2026-07-19

## Українською

Codex дозволено самостійно:

- вибрати конкретний Google-акаунт, який власник уже визначив для поточної задачі;
- виконати Google sign-in;
- використати пароль або інші дані, вже збережені штатним password manager;
- пройти OAuth consent для цього конкретного погодженого акаунта.

Значення паролів, токенів, cookies, ключів або recovery-кодів не можна копіювати в GitHub, відповіді, логи чи checkpoint.

## English

Codex may autonomously select the specific Google account already identified by the owner for the current task, complete Google sign-in, use credentials already stored by the system password manager, and complete OAuth consent for that approved account.

Password, token, cookie, key, and recovery-code values must never be copied into GitHub, responses, logs, or checkpoints.
